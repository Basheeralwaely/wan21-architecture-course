class SelfAttention(nn.Module):
    def __init__(self, dim: int, num_heads: int, eps: float = 1e-6):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = dim // num_heads
        self.q = nn.Linear(dim, dim)
        self.k = nn.Linear(dim, dim)
        self.v = nn.Linear(dim, dim)
        self.o = nn.Linear(dim, dim)
        self.norm_q = RMSNorm(dim, eps=eps)
        self.norm_k = RMSNorm(dim, eps=eps)
        self.attn = AttentionModule(self.num_heads)

    def forward(self, x, freqs):
        q = self.norm_q(self.q(x))     # q normalized BEFORE RoPE
        k = self.norm_k(self.k(x))     # k normalized BEFORE RoPE
        v = self.v(x)                   # v is NOT normalized
        q = rope_apply(q, freqs, self.num_heads)
        k = rope_apply(k, freqs, self.num_heads)
        x = self.attn(q, k, v)
        return self.o(x)
```

class CrossAttention(nn.Module):
    def __init__(self, dim: int, num_heads: int, eps: float = 1e-6, has_image_input: bool = False):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = dim // num_heads
        self.q = nn.Linear(dim, dim)
        self.k = nn.Linear(dim, dim)
        self.v = nn.Linear(dim, dim)
        self.o = nn.Linear(dim, dim)
        self.norm_q = RMSNorm(dim, eps=eps)
        self.norm_k = RMSNorm(dim, eps=eps)
        self.has_image_input = has_image_input
        self.attn = AttentionModule(self.num_heads)
        if has_image_input:
            self.k_img = nn.Linear(dim, dim)
            self.v_img = nn.Linear(dim, dim)
            self.norm_k_img = RMSNorm(dim, eps=eps)

    def forward(self, x: torch.Tensor, y: torch.Tensor):
        if self.has_image_input:
            img = y[:, :257]            # CLIP: 1 CLS + 256 patch tokens
            ctx = y[:, 257:]            # T5 text tokens
        else:
            ctx = y
        q = self.norm_q(self.q(x))
        k = self.norm_k(self.k(ctx))
        v = self.v(ctx)
        x = self.attn(q, k, v)         # main text cross-attention
        if self.has_image_input:
            k_img = self.norm_k_img(self.k_img(img))
            v_img = self.v_img(img)
            y = flash_attention(q, k_img, v_img, num_heads=self.num_heads)
            x = x + y                  # image branch added
        return self.o(x)
```

**Cell 3 — Markdown: The adaLN-Zero conditioning mechanism**
Explain the concept:
- Standard adaptive layer norm (adaLN) applies scale and shift based on a conditioning signal (e.g., timestep embedding)
- adaLN-Zero adds **gating**: when gates start at zero, each block is an identity function at initialization — output == input
- This means training starts from a stable identity and gradually learns to incorporate the attention/FFN branches
- The original DiT paper achieves this via zero-initialized final linear layers
- Wan2.1 takes a different approach: a learned `modulation` parameter per block

**Cell 4 — Markdown: The GateModule — gate=0 identity**
Explain GateModule: `x + gate * residual`. When gate=0, output is just x (identity). When gate=1, output is x + residual (full residual connection). Cite: `Source: diffsynth/models/wan_video_dit.py, line 190`.

**Cell 8 — Markdown: Wan2.1 vs DiT paper initialization**
Critical distinction (Pitfall 4):
- **DiT paper**: Gates are zero-initialized via zero-init of the final linear layer -> blocks start as exact identity
- **Wan2.1**: Uses `torch.randn(1, 6, dim) / dim**0.5` -> blocks start as NEAR-identity (small random perturbation) but NOT exact identity
- The gate=0 identity property is still the motivating design principle — it's why adaLN-Zero works. Wan2.1 just uses a softer version of the initialization.
