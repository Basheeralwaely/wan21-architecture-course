```python
class CausalConv3d(nn.Conv3d):
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self._padding = (self.padding[2], self.padding[2], self.padding[1],
                         self.padding[1], 2 * self.padding[0], 0)
        self.padding = (0, 0, 0)

    def forward(self, x, cache_x=None):
        padding = list(self._padding)
        if cache_x is not None and self._padding[4] > 0:
            cache_x = cache_x.to(x.device)
            x = torch.cat([cache_x, x], dim=2)
            padding[4] -= cache_x.shape[2]
        x = F.pad(x, padding)
        return super().forward(x)
```

AttentionBlock (lines 304-342):
```python
def forward(self, x):
    identity = x
    b, c, t, h, w = x.size()
    x = rearrange(x, 'b c t h w -> (b t) c h w')
    x = self.norm(x)
    q, k, v = self.to_qkv(x).reshape(b*t, 1, c*3, -1).permute(0,1,3,2).contiguous().chunk(3, dim=-1)
    x = F.scaled_dot_product_attention(q, k, v)
    x = x.squeeze(1).permute(0, 2, 1).reshape(b*t, c, h, w)
    x = self.proj(x)
    x = rearrange(x, '(b t) c h w-> b c t h w', t=t)
    return x + identity
```
- norm = RMS_norm(dim, channel_first=True, images=True) -- images=True for 4D (bt,c,h,w)
- to_qkv = Conv2d(dim, dim*3, 1); proj = Conv2d(dim, dim, 1), zero-initialized

Cell 2 (markdown): `## 1. CausalConv3d -- Causal Temporal Padding for Video`
Prose explaining: video generation needs convolutions that don't look at future frames. Standard Conv3d pads symmetrically (sees both past and future). CausalConv3d applies asymmetric padding: prepend `2 * temporal_pad` zero frames on the LEFT, zero frames on the right. Result: each output frame depends only on current and past input frames.
Reference: `diffsynth/models/wan_video_vae.py, lines 33-52`

Cell 3 (markdown): ASCII diagram per D-01. EXACT content:
```
### Asymmetric Temporal Padding

CausalConv3d Temporal Padding  (kernel_t=3, padding=1 -> temporal_pad=1)
=========================================================================
Input frames:   [f-2] [f-1] [ f ]   <- T=3 real frames
                  |     |     |
F.pad prepend:  [  0] [  0] [f-2] [f-1] [f] <- 2*temporal_pad = 2 zeros prepended
                  |     |     |     |     |
Conv3d k=3:          [window-0]          = uses f-2, f-1, f  (no future!)
                            [window-1]   = uses f-1,  f, 0  <- right pad = 0
No right padding: output T = input T (shape preserved)
Formula: _padding = (right_W, right_W, bot_H, bot_H, 2*t_pad, 0)
         for padding=1: (1, 1, 1, 1, 2, 0)
```

Cell 6 (markdown): `## 2. RMS_norm -- VAE vs DiT Normalization Convention`
Brief note: both VAE and DiT use RMSNorm, but with different broadcasting. VAE tensors are 5D `[B,C,T,H,W]` (channel-first), DiT tensors are 3D `[B,S,dim]` (channel-last). Include comparison table:

| Property | VAE `RMS_norm` | DiT `WanRMSNorm` (NB-01) |
|----------|---------------|--------------------------|
| Source | `wan_video_vae.py:55` | `wan_video_dit.py:101` |
| Normalize axis | `dim=1` (channel) | `dim=-1` (feature) |
| Input format | `[B, C, T, H, W]` | `[B, S, dim]` |
| Gamma shape | `[C, 1, 1, 1]` (`images=False`) | `[dim]` |
| API | `RMS_norm(dim, images=False)` | `WanRMSNorm(dim, eps)` |

Cell 11 (markdown): `## 4. AttentionBlock -- Per-Frame Spatial Self-Attention`
Prose explaining: AttentionBlock applies single-head self-attention WITHIN each frame (spatial positions attend to each other). Temporal dimension is collapsed into the batch dimension, so frames do NOT attend to each other. This is 2D spatial attention on 5D video data.
Key detail: `to_qkv = Conv2d(dim, dim*3, 1)` projects channels to Q, K, V. `proj = Conv2d(dim, dim, 1)` is zero-initialized (like adaLN-Zero in NB-05, the block starts as identity).
Reference: `diffsynth/models/wan_video_vae.py, lines 304-342`

Cell 12 (code): AttentionBlock forward trace:
```python

print("AttentionBlock(dim=8)")

print(f"  proj.bias all zeros:   {(ab.proj.bias == 0).all().item()}")    # True

    out = ab(x)

# Step-by-step shape trace:
# 1. rearrange 'b c t h w -> (b t) c h w': [1,8,4,8,8] -> [4,8,8,8]  (T folded into batch)
# 2. norm: [4,8,8,8] -> [4,8,8,8]
# 3. to_qkv: [4,8,8,8] -> [4,24,8,8]  (dim*3 = 24)
# 4. reshape+permute+chunk: q,k,v each [4,1,64,8]  (h*w=64, single head, dim=8)
# 5. scaled_dot_product_attention: [4,1,64,8]
# 6. reshape back: [4,8,8,8]
# 7. proj: [4,8,8,8] -> [4,8,8,8]  (zero-init: initially identity residual)
# 8. rearrange '(b t) c h w -> b c t h w': [4,8,8,8] -> [1,8,4,8,8]
# 9. return x + identity

assert out.shape == (1, 8, 4, 8, 8), f"Expected (1,8,4,8,8), got {out.shape}"
print(f"AttentionBlock: {list(x.shape)} -> {list(out.shape)}")
```

Cell 14 (markdown): `## Summary`
Key takeaways:
- **CausalConv3d**: wraps nn.Conv3d with asymmetric temporal padding `(right_W, right_W, bot_H, bot_H, 2*t_pad, 0)`. Preserves T, H, W dimensions. Ensures each output frame sees only current and past input frames.
- **RMS_norm**: VAE uses channel_first convention (`dim=1`, gamma `[C,1,1,1]`), different from DiT's channel_last (`dim=-1`, gamma `[dim]`).
- **ResidualBlock**: pipeline of `RMSNorm->SiLU->CausalConv3d->RMSNorm->SiLU->Dropout->CausalConv3d` with Identity or 1x1 skip connection. The building block stacked in every encoder/decoder level (NB-10, NB-11).
- **AttentionBlock**: per-frame spatial self-attention (T collapsed into batch). Single-head. Zero-initialized output projection for identity-at-init behavior. Used in encoder/decoder middle blocks.
