Cell 4 (markdown): `## 2. Resample Module -- Spatial and Temporal Downsampling`
Prose per D-05 explaining: Resample handles the spatial reduction between encoder levels. Two modes for downsampling:
- `downsample2d`: ZeroPad2d(0,1,0,1) + Conv2d(dim, dim, 3, stride=(2,2)) -- halves H and W only
- `downsample3d`: same spatial path + `time_conv = CausalConv3d(dim, dim, (3,1,1), stride=(2,1,1))` -- adds temporal stride
IMPORTANT: `time_conv` only runs when `feat_cache is not None` (production streaming). In demo cells (no feat_cache), `downsample3d` behaves identically to `downsample2d`.
Reference: `diffsynth/models/wan_video_vae.py, lines 82-196`

Cell 9 (markdown): `## 4. Reparameterization -- Sampling from the Latent Distribution`
Code-focused prose per D-06: the encoder outputs `z_dim*2` channels. These are split into mu and log_var via `.chunk(2, dim=1)`. The reparameterization trick: instead of sampling z directly from N(mu, sigma^2) (which blocks gradient flow), compute `z = mu + eps * exp(0.5 * log_var)` where `eps ~ N(0, 1)`. Gradients flow through mu and log_var (encoder outputs) because eps is treated as a constant. No ELBO/loss theory.
Reference: `diffsynth/models/wan_video_vae.py, lines 1036-1039 (reparameterize), line 1001 (mu/log_var split)`

Cell 10 (code): Reparameterization demo per D-06:
```python
# Source: diffsynth/models/wan_video_vae.py, lines 1036-1039
def reparameterize(mu, log_var):
    std = torch.exp(0.5 * log_var)  # std = sqrt(var) = exp(0.5 * log(var))
    eps = torch.randn_like(std)     # sample from N(0,1) -- NOT a function of parameters
    return eps * std + mu           # z ~ N(mu, std^2) -- gradients flow through mu and std

# Demonstrate mu/log_var split from encoder output:
# Source: wan_video_vae.py, line 1001: mu, log_var = self.conv1(out).chunk(2, dim=1)
enc_output = z  # [1, 8, 5, 2, 2] -- z_dim*2 = 8 channels from encoder head
mu, log_var = enc_output.chunk(2, dim=1)  # split along channel axis
print(f"Encoder output:  {list(enc_output.shape)}")  # [1, 8, 5, 2, 2]
print(f"mu:              {list(mu.shape)}")           # [1, 4, 5, 2, 2]
print(f"log_var:         {list(log_var.shape)}")      # [1, 4, 5, 2, 2]

latent_z = reparameterize(mu, log_var)
print(f"Latent z:        {list(latent_z.shape)}")     # [1, 4, 5, 2, 2]
assert latent_z.shape == mu.shape  # z has same shape as mu
assert latent_z.shape == (1, 4, 5, 2, 2)

# The scale factor (VideoVAE_ line 1004): z = z * self.scale_factor
# In production scale_factor is a learned scalar that normalizes the latent range
print("In production: z = z * scale_factor (learned normalization)")
```

Cell 2 (markdown): `## 1. Decoder3d -- Mirroring the Encoder`
Prose per D-07: "Decoder3d mirrors Encoder3d in reverse." Start with this framing. Then focus on what's DIFFERENT:
1. **Resample upsample mode**: `upsample3d` uses `Upsample(scale_factor=(2,2)) + Conv2d(dim, dim//2, 3, padding=1)` -- the Conv2d HALVES channels (unlike downsample which preserves them)
2. **Extra ResBlock per level**: encoder uses `num_res_blocks`, decoder uses `num_res_blocks + 1`
3. **in_dim halving**: at levels 1, 2, 3, the decoder halves `in_dim` to compensate for the channel halving in the preceding upsample Resample
4. **Reversed dim_mult**: decoder processes levels from highest channels to lowest

Include mirror comparison table per D-07:

| Sub-module | Type | Encoder Mirror? | Key Difference |
|-----------|------|----------------|----------------|
| `conv1` | `CausalConv3d(z_dim, dims[0], 3, p=1)` | Yes | Input is z_dim latent, not 3-channel RGB |
| `middle` | `Sequential(ResBlock, AttnBlock, ResBlock)` | Yes -- identical | Applied BEFORE upsamples (encoder: AFTER downsamples) |
| `upsamples` | `nn.Sequential(...)` | Mirror | `num_res_blocks + 1` ResBlocks per level |
| `head` | `Sequential(RMS_norm, SiLU, CausalConv3d(dim->3))` | Yes | Output is 3-channel RGB |

Cell 4 (markdown): `### in_dim Halving -- The Decoder's Channel Compensation`
Prose explaining Pitfall 2: the decoder's `dims` list for dim=8, dim_mult=[1,2,4,4] is `[32, 32, 16, 8, 8]`. But at levels 1, 2, and 3, the code does `if i == 1 or i == 2 or i == 3: in_dim = in_dim // 2`. This compensates for upsample3d's Conv2d outputting `dim//2` channels. Without this halving, ResBlocks would receive the wrong channel count.

```python
# Decoder3d.__init__ loops over dims pairs and applies in_dim halving:
dim = 8
dim_mult = [1, 2, 4, 4]
dims = [dim * m for m in dim_mult] + [dim * dim_mult[-1]]  # [8, 16, 32, 32, 32]
dims = dims[::-1]  # reversed: [32, 32, 32, 16, 8]
# Note: decoder reverses the dims from encoder

print("Decoder level-by-level in_dim computation:")
print(f"dims (reversed): {dims}")
for i, (in_dim_orig, out_dim) in enumerate(zip(dims[:-1], dims[1:])):
    in_dim = in_dim_orig
    if i == 1 or i == 2 or i == 3:
        in_dim = in_dim // 2  # compensate for Resample channel halving
    print(f"  Level {i}: in_dim={in_dim_orig} -> {in_dim} (halved={i in [1,2,3]}), out_dim={out_dim}")
    # Level 0: in_dim=32, out_dim=32 (no halving)
    # Level 1: in_dim=32 -> 16, out_dim=16 (halved -- Resample halved channels from 32 to 16)
    # Level 2: in_dim=32 -> 16, out_dim=8  (halved -- Resample halved channels from 32 to 16)
    # Level 3: in_dim=16 -> 8, out_dim=8   (halved -- Resample halved channels from 16 to 8)
```

```
VAE patchify vs DiT patchify
=============================
VAE patchify (einops, no params):
  [B, 16, F, H, W] -> rearrange "b c f (h q) (w r) -> b (c r q) f h w"
  -> [B, 64, F, H/2, W/2]    Still a 5D video tensor!
  Spatial pixels -> channels  (2x2 spatial patch becomes 4 extra channels)

DiT patchify (Conv3d + rearrange, has params):
  [B, 48, F, H, W] -> Conv3d(k=(1,2,2), s=(1,2,2)) -> [B, 1536, F, H/2, W/2]
  -> rearrange "b c f h w -> b (f h w) c" -> [B, F*H/2*W/2, 1536]
  Spatial patches + channels -> flat TOKEN SEQUENCE for DiT
```
