**Cell 13 (markdown) -- Section 4: Denoising Loop (D-03):**
This is the KEY teaching section. Write for someone encountering flow matching for the first time (per user's specific request in CONTEXT.md `<specifics>`). Cover:

1. **What is flow matching?** In standard diffusion (DDPM), the model predicts noise. In flow matching, the model predicts velocity -- the direction from noise to clean data. The training target is `v = data - noise` (not epsilon/noise). At inference, we follow this velocity field from pure noise toward the clean image.

2. **The denoising schedule:** FlowMatchScheduler creates a sequence of sigma values (1.0 -> ~0.093 for 50 steps with shift=5.0). sigma=1.0 means pure noise, sigma=0 means clean image. Each step reduces sigma.

3. **The step formula:** `prev_sample = sample + velocity * (sigma_next - sigma_current)`. Since `sigma_next < sigma_current`, the delta is negative. The velocity points from noise toward data, so multiplying by a negative delta moves the sample toward the clean manifold.

4. **Classifier-Free Guidance (CFG):** Run the model twice -- once with the real text prompt (positive prediction), once with empty/negative prompt (negative prediction). Combine: `noise_pred = neg + cfg_scale * (pos - neg)`. `cfg_scale=1.0` means no guidance (just positive). `cfg_scale=7.5` amplifies the difference, steering generation toward the prompt.

### Key Takeaways

- **Pipeline composition**: WanVideoPipeline orchestrates four model components: T5 text encoder (context), CLIP image encoder (visual features), VAE (video <-> latent space), and DiT (denoising in latent space)
- **48-channel input**: The Fun-Control DiT variant concatenates noise latents (16ch) + control latents (16ch) + reference latents (16ch) into a 48-channel input processed by the patch embedding Conv3d
- **Flow matching denoising**: Unlike DDPM noise prediction, flow matching predicts velocity (direction from noise to data). Each step applies `latents += velocity * (sigma_next - sigma_current)` where the negative delta moves toward the clean image
- **Classifier-Free Guidance**: Two DiT passes (positive + negative prompt) are combined to steer generation toward the text prompt
- **Scale**: The full pipeline is ~8.0B parameters, dominated by T5 (5.68B, 71%)
